# TAS-1: Training Attribution Standard

TAS-1 is an open standard for **cryptographic attribution of work in
decentralized AI training runs**. It specifies the bytes on the wire and
the leaf semantics that let any verifier — on-chain contract, off-chain
auditor, zk-circuit, or fraud-proof challenger — confirm that a specific
contributor produced a specific gradient artifact in a specific training
cycle.

This v1.0 covers compute (gradient) attribution. Data and capital
attribution are anticipated as v1.x minor extensions using the same
primitives.

## Status

**Draft v0.1.** The specification number `1` is reserved. The byte
formats and leaf encodings are stable. Test vectors are generated by
the conformance suite against the reference implementation.

## Why TAS-1 exists

Decentralized model training has produced its first credible empirical
result — Templar's Covenant-72B (March 2026). But every project in the
space — Templar, Macrocosmos IOTA, Pluralis, Gensyn, Kardashev —
re-invents:

- A gradient identity scheme
- A contributor-to-gradient binding
- An on-chain attribution record
- Replay protection

TAS-1 formalizes the smallest possible surface that solves these
problems. It is intentionally agnostic to:

- Whether the model is open or closed
- How contribution weight is derived (loss-delta scoring, validator
  quorum, ZK-proven training, anything else)
- Whether monetization is token issuance, inference revenue split, or
  no monetization
- What chain hosts the attribution records (EVM canonical;
  byte-equivalent implementations on Substrate / Solana / Cosmos
  conform)

## Conformance Levels

- **Level A (Basic Attribution).** Implements §4.1–§4.5 byte-for-byte;
  off-chain integrity is implementer's choice.
- **Level B (Signed Attribution).** Level A + EIP-191 contributor
  signatures REQUIRED at the aggregator boundary. **Recommended for
  new implementations.**

See [`spec/v1.md`](spec/v1.md) §5 for the full conformance criteria.
